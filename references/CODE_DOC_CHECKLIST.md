# Code Documentation QA Checklist

Use this checklist when editing in-code documentation.
It is a fast pass for public APIs. Keep notes as you go.
If an item does not apply, mark it and move on.

## Coverage
- [ ] Public exports have doc comments.
- [ ] Non-obvious behavior is documented.
- [ ] Examples exist for multi-step usage.

## Content quality
- [ ] Summaries use active verbs.
- [ ] Parameters and return values are documented.
- [ ] Errors and edge cases are documented.

## Accuracy
- [ ] Docs match the implementation.
- [ ] Any constraints or invariants are explicit.
